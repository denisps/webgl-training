# Development Guide

Make a clean code/modular architecture with descriptive function/object/variable names instead of relying on comments, and don't use any external dependencies.
Make the inference code not depend on the training code.
Strive to make all the functions and modules pure and avoid module/class inter dependencies.
Be concise and humble.
Don't exaggerate or overstate the achievements or architecture.
All the inference will be happening in the shedder.
Make tests, but only for important framework functions to verify that they are working as intended.
It should be a real working implementation, not just a stub with fake functions to be implemented later.
